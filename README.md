# VHDL Digital Clock with Adjustment Mode

A feature-rich digital clock implemented in VHDL, designed for the **Terasic DE1-SoC (Cyclone V)**.  
This design features a real-time clock, a variable speed mode for testing, and an interactive state machine for manually setting the time.

---

## 🚀 Features

### Real-Time Counting
Tracks **Hours, Minutes, and Seconds**.

### Time Adjustment Mode
A **7-state Finite State Machine (FSM)** allows users to set each digit of the clock individually:

- Seconds Units
- Seconds Tens
- Minutes Units
- Minutes Tens
- Hours Units
- Hours Tens

### Variable Speed (Simulation/Test)
Includes a **SPEED_FAC** input to accelerate the clock, allowing faster verification of minute and hour transitions during testing.

### SSD Integration
Uses a custom **utils package** containing an `integer_to_SSD` function to drive all six **7-segment displays** on the DE1 board.

### Synchronous Design
Clean, reset-aware synchronous logic targeting the **50 MHz onboard oscillator**.

---

## 🕹️ Interface & Controls

| Port | Type | Description |
|-----|-----|-------------|
| **Clk / Rst** | Input | 50 MHz clock and asynchronous reset (active low) |
| **ADJUST** | Input | Toggle switch to enter or exit Time Setting mode |
| **ENTER_VALUE** | Input | Key used to confirm a digit and move to the next state |
| **NEW_VALUE** | Input | 4-bit input (switches) defining the value of the current digit |
| **SPEED_FAC** | Input | 2-bit input used to divide the 50 MHz clock for faster visualization |

---

## 🧠 State Machine Logic (Adjust Mode)

When **ADJUST** is high, the clock enters a sequential state machine for setting the time.

**States**

- **IDLE** – Waiting for start
- **SU** – Set Seconds Units
- **ST** – Set Seconds Tens
- **MU** – Set Minutes Units
- **MT** – Set Minutes Tens
- **HU** – Set Hours Units
- **HT** – Set Hours Tens

---

## 📐 Clock Division Math

The default **1 Hz tick** is generated by comparing the **50 MHz clock** against a configurable limit.

Normal operation:

\[
Limit_{Normal} = 50,000,000 \text{ cycles}
\]

Fast simulation modes:

\[
Limit_{Fast} = \frac{f_{clk}}{2048}
\]

\[
Limit_{Fast} = \frac{f_{clk}}{4096}
\]

\[
Limit_{Fast} = \frac{f_{clk}}{8192}
\]

## 🛠️ How to Run

### Simulation
Use **GHDL** with a testbench to verify FSM transitions and timing behavior.

### Synthesis
Open the **`.qpf` project file** in **Quartus II 13.0 SP1**.

### Deployment
Program the generated **`.sof` file** to the **DE1-SoC FPGA**.

Ensure **`utils.vhd`** is included in the project, as it contains the **SSD driver utilities**.
